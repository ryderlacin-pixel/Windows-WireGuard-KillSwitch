# v13.3 — WebRTC Leak Guard

Adds system-level browser WebRTC leak protection on top of the existing firewall/DNS/IPv6 kill switch.

## What it does

- **Chrome / Edge / Brave:** HKLM enterprise policies
  - `WebRtcIpHandlingPolicy` = `default_public_interface_only`
  - `WebRtcLocalhostCandidateAllowed` = `0`
- **Firefox (if installed):** `distribution\policies.json` with locked ICE preferences
- **`webrtc-leak-guard.ps1`:** Re-applied on every repair cycle and stored in the anti-tamper guard vault

## Notes

- Browsers may need a restart for policies to take effect.
- WebRTC protection is browser-layer; it complements (does not replace) the tunnel kill switch.
- Non-Chromium browsers (Opera, Vivaldi, etc.) are not covered unless they honor Chromium policies.
