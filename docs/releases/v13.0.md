# v13.0 — Ultimate Fail-Open

Production release focused on **never cutting live internet** unless the tunnel is provably down after extended debounce.

## Policy

- **SafeToOpen** = tunnel RUNNING + TCP 2-of-3 (1.1.1.1, 1.0.0.1, 8.8.8.8:443). DNS does **not** gate internet open.
- **BootGrace** = 180 seconds after boot/install — no blocks from any layer.
- **UnbrickUntil** = 30 minutes after kurtar2 — no re-block.
- **Block tunnel-down** only after 5 consecutive checks (~10s).
- **Block zombie** only after 15 consecutive checks (~30s).
- **Repair fail-open** on zombie — defers blocking to monitor.
- **Watchdog graduated**: gentle unbrick → kurtar2 only after 5 unhealthy minutes.
- **Auto kurtar2** never turns firewall OFF (manual kurtar2 only).

## Rescue

- `C:\WireGuard\kurtar.bat` → `kurtar2.ps1` (full unbrick, Downloads-compatible logic)
- `resume-after-unbrick.ps1` restores tasks after 30 min cooldown

## Verify (non-disruptive)

```powershell
.\scripts\safe-live-verify.ps1
```

Do **not** run `race-recovery-test.ps1` or `ultimate-stress-test.ps1` on a live machine without `-ConfirmDisruptsInternet`.
