# v15.2.7 - Post-install internet protection

**Status:** Production - fixes internet brick after install.

## Problem

After install, DNS locked to `127.0.0.1` while the tunnel dropped. Monitor engaged kill-switch blocks. Users lost internet and had to manually delete `C:\WireGuard` files.

## Fixes

| Area | Change |
|------|--------|
| Post-install grace | 15-minute `PostInstallGraceUntil` - monitor will not block outbound during grace |
| DNS lockdown guard | Refuses to lock DNS unless tunnel RUNNING, dnscrypt listening, **and** internet TCP probe passes |
| Deferred privacy guards | `Invoke-DeferredPrivacyGuards` requires `Test-SafeToOpen` (tunnel + internet) |
| Tunnel reinstall | Backup/restore `wgcf-profile.conf` before `uninstalltunnelservice`; soft `sc start` first |
| Guard vault | `wgcf-profile.conf` backed up to `C:\ProgramData\WGKillSwitchGuard` |
| Console formatting | Unicode em-dashes replaced with ASCII `-` in user-facing messages |

## Recovery (instead of deleting files)

Run as Administrator:

```bat
emergency-reset.bat
```

Or: `.\scripts\emergency-reset.ps1 -NoPause`
