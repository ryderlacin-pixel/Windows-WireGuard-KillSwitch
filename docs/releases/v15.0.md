# v15.0 — Strong Privacy Stack

Builds on v14.0 kill switch + dnscrypt + Tor + leak-sentinel. Adds **system-wide DNS lock**, network privacy hardening, stricter dnscrypt policy, and a sensitive-browsing launcher.

## What changed

### System DNS lock (`dns-lockdown-guard.ps1`)

- All **connected** adapters → `127.0.0.1` via `netsh` (no hanging WMI cmdlets)
- `EnableAutoDoh=0`, `DisableSmartNameResolution=1`
- **Safety gate:** refuses to lock unless dnscrypt is RUNNING on `127.0.0.1:53` (prevents internet brick)

### Network privacy (`network-privacy-guard.ps1`)

- LLMNR off (`EnableMulticast=0`)
- NetBIOS off per interface (registry `NetbiosOptions=2`)
- Wi-Fi random MAC policy enabled

### Stricter dnscrypt

- `require_nolog = true`
- `server_names = ['quad9-dnsovertls']` only (Cloudflare removed)
- `KS-Dnscrypt-EXE` firewall rule allows upstream when tunnel is healthy

### Tor + sensitive mode

- Extended `user.js`: `privacy.firstparty.isolate`, referer trimming
- `sensitive-mode.ps1` + desktop shortcut **Hassas-Tarama.lnk**
- Tor still optional; start Tor Browser manually for sensitive use

### leak-sentinel v15

- System DNS check via `netsh` (not `Get-DnsClientServerAddress`)
- Raw `8.8.8.8` UDP probe is informational when firewall DNS block is off (no false crisis)
- Crisis only on real issues: system DNS leak, dnscrypt down, LLMNR on

### Recovery

- `scripts/restore-full-stack.ps1` — tunnel + tasks + NSSM + v15 sync + audits in one shot
