# v15.0 — Strong Privacy Stack

Builds on v14.0 kill switch + dnscrypt + Tor + leak-sentinel. Adds **system-wide DNS lock**, network privacy hardening, stricter dnscrypt policy, and a sensitive-browsing launcher.

## Anonymity tiers (honest)

| Mode | Command | Cost | Anonymity (honest) |
|------|---------|------|-------------------|
| **Default WARP** (recommended) | `.\install.ps1` | Free | ~7.5–8/10 — Cloudflare is the VPN operator; **excellent leak/DNS/kill-switch protection** |
| Sensitive (Tor) | `Hassas-Tarama.lnk` / `sensitive-mode.ps1` | Free | Higher for high-risk browsing only — use when you need more than WARP |
| Paid WireGuard VPN (optional) | `.\install.ps1 -CustomConfig provider.conf` | Monthly fee | ~8.5–9.5/10 — only if you can afford a no-log provider long-term |

**Default path:** Run `.\install.ps1` with free WARP. No signup, no monthly bill. v15 gives you STRONG leak protection (DNS lock, encrypted DNS, kill switch, network hardening) — that is where most users get the biggest win.

**When to add Tor:** Use **Hassas-Tarama** / `sensitive-mode.ps1` for sessions where you need stronger anonymity than WARP alone — still free.

**Paid VPN (optional, not required):** If you later get a no-log WireGuard provider (Mullvad, IVPN, ProtonVPN, etc.), swap in with `-CustomConfig` and keep the same v15 stack. Skip this if a monthly subscription is not realistic — WARP + Tor for sensitive use is a solid free setup.

## What changed

### System DNS lock (`dns-lockdown-guard.ps1`)

- All **connected** adapters → `127.0.0.1` via `netsh` (no hanging WMI cmdlets)
- `EnableAutoDoh=0`, `DisableSmartNameResolution=1`
- **Safety gate:** refuses to lock unless dnscrypt is RUNNING on `127.0.0.1:53` (prevents internet brick)

### Network privacy (`network-privacy-guard.ps1`)

- LLMNR off (`EnableMulticast=0`)
- NetBIOS off per interface (registry `NetbiosOptions=2`)
- Wi-Fi random MAC policy enabled

### Stricter dnscrypt

- `require_nolog = true`
- `server_names = ['quad9-dnsovertls']` only (Cloudflare removed)
- `KS-Dnscrypt-EXE` firewall rule allows upstream when tunnel is healthy

### Tor + sensitive mode

- Extended `user.js`: `privacy.firstparty.isolate`, referer trimming
- `sensitive-mode.ps1` + desktop shortcut **Hassas-Tarama.lnk**
- Tor still optional; start Tor Browser manually for sensitive use

### leak-sentinel v15

- System DNS check via `netsh` (not `Get-DnsClientServerAddress`)
- Raw `8.8.8.8` UDP probe is informational when firewall DNS block is off (no false crisis)
- Crisis only on real issues: system DNS leak, dnscrypt down, LLMNR on

### Recovery

- `scripts/restore-full-stack.ps1` — tunnel + tasks + NSSM + v15 sync + audits in one shot

## Honest limits

- **WARP mode (default):** Cloudflare is your VPN operator — strong leak/DNS/kill-switch protection; moderate exit anonymity. This is the intended free setup.
- **Tor (sensitive mode):** stronger for high-risk browsing; slower; use only when needed
- **Paid VPN (optional):** anonymity depends on provider quality — not required for a strong privacy stack
- Privacy tier **STRONG** = leak/tracking (~8.5–9 / ~8–8.5); does not replace threat modeling
- System DNS lock requires dnscrypt healthy — guard defers if not
- ISP still sees encrypted VPN traffic; logged-in browsers deanonymize regardless of VPN
- Tor: use only for sensitive sessions; operator-specific threat models apply
