# v14.0 — DNS Encryption + Tor Hardening + Leak Sentinel

Builds on v13.5 kill switch and privacy hardening. Adds a fourth OSS component (dnscrypt-proxy) plus Tor Browser policy hardening and read-only leak monitoring.

## What changed

### dnscrypt-proxy (DNS leak stack)

- Downloads **dnscrypt-proxy 2.1.7** with SHA256 manifest verification
- Listens on `127.0.0.1:53`, service **WG-DnscryptProxy**
- WireGuard config `DNS = 127.0.0.1` (tunnel reload required)
- `dnscrypt-guard.ps1` re-applied by repair

### Tor Browser (optional)

- Detects existing Tor Browser install paths (no auto-download)
- Applies hardened `user.js` (SOCKS remote DNS, WebRTC off, HTTPS-only)
- `tor-connectivity-monitor.ps1` probes SOCKS `127.0.0.1:9150` when Tor is running
- **Sensitive browsing only** — WARP/Cloudflare still sees tunnel entry

### leak-sentinel

- Read-only UDP probe to `8.8.8.8:53` and dnscrypt service check
- **Never** calls `Enable-Block` or changes firewall rules
- Crisis log: `C:\WireGuard\leak-crisis.jsonl`, registry `LeakState`

## Phased upgrade

```powershell
# Preflight (port 53 check)
.\scripts\tor-preflight.ps1

# Phase 1: DNS only (safe on live system)
.\install.ps1 -DnsLeakUpgradeOnly -NoPause
.\scripts\leak-audit.ps1

# Phase 2: Tor hardening (install Tor Browser manually first)
.\install.ps1 -TorUpgradeOnly -NoPause
.\scripts\tor-audit.ps1

# Phase 3: Full v14 privacy (dnscrypt + Tor + leak-sentinel + v13.5 privacy)
.\install.ps1 -FullPrivacyUpgrade -NoPause

# Full reinstall
.\install.ps1 -NoPause
```

After DNS change: **restart WireGuard tunnel** (deactivate/activate in WireGuard UI).
