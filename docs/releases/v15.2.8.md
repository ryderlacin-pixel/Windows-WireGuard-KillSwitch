# v15.2.8 - Install must not brick internet

## Root causes found

1. **DNS locked to 127.0.0.1** while tunnel dropped after install
2. **emergency-reset** ran `netsh winsock reset` + `ip reset` without reboot (breaks live session)
3. User ran emergency-reset multiple times in panic (log shows 3 runs in 20s)
4. DNS lock applied before stack was stable for 30s

## Fixes

| Fix | Detail |
|-----|--------|
| Safe emergency-reset | No winsock/IP/firewall full reset by default (`-DeepReset` optional) |
| emergency-reset | Stops/disables WG tasks, restores DHCP DNS, sets 30min grace |
| Fail-open DNS | Monitor/watchdog restore DHCP DNS when tunnel down |
| Deferred DNS lock | Requires 3x SafeToOpen over 30s before applying guards |
| Monitor delay | 45s delay after install before monitor starts |
| STEP 18f | No false "DNS lock OK" while install lock active |

## If internet dies after install

**Do NOT delete C:\WireGuard.** Run once as Admin:

```bat
emergency-reset.bat
```

Reboot once. Do not click it multiple times.
