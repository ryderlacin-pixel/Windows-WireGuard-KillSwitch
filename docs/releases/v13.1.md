# v13.1 — Monitor-Only Block Authority

Builds on v13.0 fail-open policy with stricter **single-writer** block semantics: only `monitor.ps1` may enable outbound blocks.

## Changes from v13.0

- **Repair / GPO / SVC never `Enable-Block`** — repair only opens internet when healthy; never races monitor to block.
- **Monitor startup always fail-open** — unhealthy boot waits for debounce in main loop, never blocks at startup.
- **Recovery loop never re-blocks** — tunnel reinstall retries without re-applying blocks mid-recovery.
- **`resume-after-unbrick.ps1` sets BootGrace** — 180s fail-open after kurtar2 cooldown ends.
- **Tunnel dual-check** — service RUNNING **and** WireGuard adapter Up (reduces sc.exe flicker false positives).

## Policy (unchanged from v13.0)

- **SafeToOpen** = tunnel (service+adapter) + TCP 2-of-3. DNS does not gate open.
- **BootGrace** = 180s after boot/install/resume.
- **UnbrickUntil** = 30 min after kurtar2.
- **Block tunnel-down** after 5× (~10s); **zombie** after 15× (~30s).
