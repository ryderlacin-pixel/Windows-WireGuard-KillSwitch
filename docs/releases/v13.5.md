# v13.5 — Privacy Engineer Pass

Honest threat-model scores and expanded policy coverage on top of v13.4. Kill switch unchanged.

## What changed

### Browser (Chrome / Edge / Brave)

- Privacy Sandbox (Topics, measurement, prompts) disabled
- DNS-over-HTTPS policy forced **off** (no browser DoH bypass)
- QUIC disabled (`QuicAllowed=0`)
- Extension sideload blocked (`ExtensionInstallBlocklist=*`) — may block all extensions
- Network prediction / search suggest off

### Firefox

- `privacy.fingerprintingProtection` + existing `resistFingerprinting`
- `webgl.disabled`, `dom.webgpu.enabled=false`
- Stricter referer policy

### Windows

- Windows Error Reporting disabled
- More AppPrivacy deny defaults (location, mic, camera, clipboard history)
- Sensors disabled
- Wording: **consumer telemetry reduced (not eliminated)**

### Supply chain

- `Install-ScriptIntegrityVault` — SHA256 hashes in `HKLM:\SOFTWARE\WGKillSwitch`
- `safe-live-verify.ps1` verifies monitor/repair/privacy-guard hashes
- `-PrivacyUpgradeOnly` — refresh privacy + guard + integrity without full reinstall

## Honest scores (project scope)

| Metric | v13.4 claim | v13.5 (honest) |
|--------|-------------|----------------|
| Leak prevention | 8.5/10 | **8–8.5/10** |
| Tracking / fingerprint surface | ~9/10 | **7.5–8/10** (attack surface reduced, not eliminated) |
| General anonymity (WARP threat model) | ~9/10 | **7–8/10** |

### Out of scope (documented, not fixed)

- Tor, multi-hop, pluggable transports, traffic shaping
- Hiding traffic from **Cloudflare** (WARP terminates at Cloudflare)
- JA3/TLS fingerprint, canvas/font/timing attacks, extension fingerprint
- Full telemetry elimination (kernel, Defender, drivers, MSA sync)
- Code signing (trust repo + CI + integrity vault)
- Stealth vs forensic footprint (WMI/GPO/tasks remain visible)
